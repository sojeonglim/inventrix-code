# 코드 품질 평가

## 테스트 커버리지
- **전체**: 없음 (테스트 코드 없음)
- **단위 테스트**: 없음
- **통합 테스트**: 없음
- **E2E 테스트**: 없음

## 코드 품질 지표
- **린팅**: 미설정 (ESLint 없음)
- **코드 스타일**: 비교적 일관적이나 Prettier 등 포매터 미설정
- **문서화**: 최소한 (README.md만 존재, 코드 내 주석 없음)
- **타입 안전성**: TypeScript 사용하나 `any` 타입 다수 사용

## 기술 부채

### 심각 (P0)
1. **SQLite 프로덕션 사용** - 동시성 제한, 단일 writer, 프로덕션 부적합
2. **재고 예약 시스템 없음** - 동시 주문 시 overselling 가능 (경쟁 조건)
3. **하드코딩된 JWT Secret** - `'inventrix-secret-key-change-in-production'` 소스코드에 노출

### 높음 (P1)
4. **입력 검증 없음** - API 엔드포인트에 요청 데이터 검증 없음
5. **요청 제한 없음** - API 남용/DoS 취약
6. **에러 처리 미흡** - 일부 에러만 처리, 전역 에러 핸들러 없음
7. **CORS 무제한** - `app.use(cors())` 모든 origin 허용
8. **보안 헤더 없음** - helmet.js 등 미사용
9. **감사 로깅 없음** - 사용자 활동 추적 불가

### 보통 (P2)
10. **라우트에서 직접 DB 쿼리** - Service/Repository 계층 분리 없음
11. **프론트엔드 인라인 스타일** - CSS 모듈이나 스타일 라이브러리 미사용
12. **프론트엔드에서 `any` 타입 사용** - AdminDashboard 등에서 타입 안전성 부족
13. **비밀번호 정책 없음** - 최소 길이, 복잡성 요구사항 없음
14. **토큰 만료 처리 미흡** - 프론트엔드에서 만료된 토큰 처리 로직 없음
15. **주문 취소 시 재고 복원 없음** - 취소해도 재고가 돌아오지 않음

### 낮음 (P3)
16. **접근성 미흡** - ARIA 속성, 키보드 네비게이션 미지원
17. **반응형 디자인 미흡** - 모바일 최적화 부족
18. **Lock file 관리** - pnpm-lock.yaml 존재하나 의존성 취약점 스캔 미설정
19. **중복 상품명** - 'Tablet 10"' 상품이 시드 데이터에 2개 존재

## 패턴 및 안티패턴

### 좋은 패턴
- TypeScript 전체 사용으로 타입 안전성 기본 확보
- Monorepo 구조로 코드 관리 용이
- bcrypt 비밀번호 해싱 사용
- JWT 기반 무상태(stateless) 인증
- Prepared statements 사용 (SQL 인젝션 방지)
- EC2 배포 시 EBS 암호화, IMDSv2 강제

### 안티패턴
- **비대한 라우트(Fat Routes)**: 비즈니스 로직이 라우트 핸들러에 직접 구현 (Service 계층 없음)
- **인라인 스타일**: 프론트엔드 전체가 인라인 스타일 사용
- **하드코딩된 시크릿**: JWT secret이 소스코드에 하드코딩
- **검증 계층 없음**: 입력 검증 없이 DB에 직접 저장
- **에러 바운더리 없음**: 프론트엔드에 React Error Boundary 없음
- **로딩/에러 상태 없음**: 대부분의 페이지에서 에러 상태 처리 없음
