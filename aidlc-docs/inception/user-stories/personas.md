# 사용자 페르소나

## Persona 1: 고객 (Customer)
- **역할**: 일반 사용자
- **목표**: 상품 탐색, 주문 생성, 주문 이력 확인
- **특성**: 기술 수준 다양, 모바일/데스크톱 모두 사용, 빠른 응답과 명확한 피드백 기대
- **주요 시나리오**: 상품 검색 → 주문 → 상태 확인 → 취소/환불
- **불만 요소**: 느린 로딩, 주문 상태 불투명, 모바일에서 사용 어려움

## Persona 2: 관리자 (Admin)
- **역할**: 시스템 및 비즈니스 전체 관리자
- **목표**: 시스템 설정, 사용자 역할 관리, 상품/주문/재고 관리, 비즈니스 분석, AI 이미지 생성, 보안 모니터링
- **특성**: 기술 이해도 높음, 보안 의식 강함, 비즈니스 중심 사고, 대시보드 의존, 실시간 데이터 필요
- **주요 시나리오**: 역할 할당/변경, 상품 CRUD, 주문 상태 관리, 분석 대시보드 조회, 재고 모니터링, 보안 이벤트 모니터링
- **불만 요소**: 감사 로그 부재, 역할 관리 불가, 실시간 업데이트 없음, 재고 부족 알림 없음, 분석 데이터 제한적

## Persona 3: 직원 (Staff)
- **역할**: 내부 운영 직원 (고객 지원, 물류/재고 담당 포함)
- **목표**: 주문/상품/재고 정보 조회, 재고 수준 관리, 배송 처리
- **특성**: 고객 문의 대응, 현장 중심, 태블릿/모바일 사용 빈번, 빠른 작업 처리 필요
- **주요 시나리오**: 고객 주문 조회, 상품 정보 확인, 재고 확인/수정, 배송 상태 업데이트 (processing→shipped→delivered)
- **불만 요소**: 필요한 정보에 접근 불가, 느린 검색, 모바일 UI 부적합, 재고 변경 이력 없음
