# Frontend Mock API 테스트 케이스

> cross-unit-contracts.md 기반. Backend 완료 전 Mock으로 독립 테스트 가능.

---

## API Client

### 인증 토큰 관리
- [ ] 로그인 성공 → token 저장
- [ ] 모든 인증 API 요청에 Authorization: Bearer 헤더 자동 첨부
- [ ] 401 응답 → refresh token으로 자동 갱신 시도
- [ ] refresh 실패 → 로그인 페이지 리다이렉트
- [ ] 로그아웃 → token 삭제

### 에러 처리
- [ ] 400 → 폼 검증 에러 표시
- [ ] 403 → "권한이 없습니다" 메시지
- [ ] 404 → "찾을 수 없습니다" 메시지
- [ ] 409 → 비즈니스 에러 메시지 (재고 부족 등)
- [ ] 429 → "요청이 너무 많습니다" 메시지
- [ ] 500 → "서버 오류" 일반 메시지 (내부 정보 노출 없음)
- [ ] 네트워크 에러 → "연결할 수 없습니다" + 재시도 옵션

## SSE 연결

- [ ] 로그인 후 SSE 자동 연결
- [ ] notification 이벤트 → 토스트 표시 + 알림 벨 카운트 증가
- [ ] dashboard_update 이벤트 (admin) → 대시보드 KPI 갱신
- [ ] 연결 끊김 → exponential backoff 재연결 (1s, 2s, 4s, 8s, max 30s)
- [ ] 로그아웃 → SSE 연결 해제

## 역할별 UI 렌더링

### Navigation
- [ ] admin → 모든 메뉴 표시 (대시보드, 상품, 주문, 재고, 사용자, 분석)
- [ ] staff → 주문, 재고 메뉴만 표시
- [ ] customer → 상점, 내 주문 메뉴만 표시
- [ ] 역할 관리 메뉴 → admin만 표시

### 주문 페이지
- [ ] customer → 본인 주문만 표시, 취소 버튼 (pending만)
- [ ] admin → 전체 주문, 상태 변경 드롭다운
- [ ] staff → 전체 주문, 배송 상태 변경만 가능
