# Deployment Guide — Inventrix Infrastructure

## Prerequisites
- AWS CLI configured with appropriate credentials
- AWS CDK CLI installed (`npm install -g aws-cdk`)
- EC2 Key Pair created: `inventrix-staging-key`, `inventrix-production-key`
- Domain configured (optional, for SSL)

## Initial Provisioning

### 1. CDK Bootstrap (first time only)
```bash
cd packages/infrastructure
pnpm install
cdk bootstrap aws://ACCOUNT_ID/ap-northeast-2
```

### 2. Deploy Staging
```bash
cdk deploy --all -c env=staging -c deployerIp=YOUR_IP/32 -c alertEmail=admin@example.com
```

### 3. Set Secrets (after deploy)
```bash
aws ssm put-parameter \
  --name /inventrix/staging/JWT_SECRET \
  --type SecureString \
  --value "$(openssl rand -base64 64)" \
  --overwrite

# DATABASE_URL is auto-generated by RDS credentials in Secrets Manager
```

### 4. DNS Setup
- Point domain A record to Elastic IP from CDK output
- Add SES DKIM CNAME records (check SES console)

### 5. SSL Setup
```bash
ssh ec2-user@ELASTIC_IP
sudo certbot --nginx -d inventrix.example.com
```

### 6. Initial Application Deploy
```bash
ssh ec2-user@ELASTIC_IP
cd /var/www/inventrix
git clone https://github.com/sojeonglim/inventrix-code.git .
pnpm install --frozen-lockfile
pnpm --filter @inventrix/api build
pnpm --filter @inventrix/frontend build
sudo cp packages/infrastructure/nginx/nginx.conf /etc/nginx/nginx.conf
sudo nginx -t && sudo systemctl restart nginx
pm2 start packages/infrastructure/pm2/ecosystem.config.js --env staging
pm2 save
pm2 startup
```

### 7. DB Migration
```bash
cd /var/www/inventrix/packages/api
pnpm migrate  # or run migration SQL directly
```

## Subsequent Deployments
Handled automatically by GitHub Actions:
- **Staging**: Push to main → auto deploy
- **Production**: Manual dispatch in GitHub Actions

## Rollback
```bash
ssh ec2-user@ELASTIC_IP
cd /var/www/inventrix
git checkout HEAD~1
pnpm install --frozen-lockfile
pnpm --filter @inventrix/api build
pnpm --filter @inventrix/frontend build
pm2 reload ecosystem.config.js
```

## Destroy (cleanup)
```bash
cd packages/infrastructure
cdk destroy --all -c env=staging
```
